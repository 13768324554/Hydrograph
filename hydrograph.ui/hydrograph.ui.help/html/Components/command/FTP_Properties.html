FTP Properties

Properties for the FTP component can be viewed by Double click component on canvas. The properties contain a 'General' tab. Common and mandatory properties are present in the General tab.

General Properties

Details

  • Name - The Name field will specify the name of the component as desired by the user.
  • ID - ID field will specify unique id for every component.
  • Type - Type further categorizes the component to detailed operation that it intends to perform. Here it is of type command FTP.

Configuration

  • Protocol - Contains a drop-down of Protocols. Used to define file transfer protocols. Drop down lists values like : FTP, SFTP, AWS S3 HTTPS.
  • Host Name - Host Name used to specify the Host server IP address or localhost name. This field should be mandatory to user for FTP and SFTP Protocol.
  • Port - Port simply accepts a numeric number starting from 0 to maximum 65535.
  • Authentication - User needs to provide credentials to connect to the ftp server.
        • If user select protocol : FTP
    • Authentication Mode - Authentication Mode has One options -
        Standard Authentication - It has two fields -
        User ID - User is required to specify the User name to login to Teradata Database.
        Password - Password used to specify the password.
        • If user select protocol : SFTP
    • Authentication Mode - Authentication Mode has two options -
        Standard Authentication

        USER ID and Key

        If user select protocol : AWS S3 HTTPS
    • Authentication Mode - Authentication Mode has two options -
        AWS S3 Access Key

        AWS S3 Property File

  • Connection Time-Out - It defines the connection time out in milliseconds.
  • Number of Retries - It defines the no of tries to establish the connection.
  • Operation Configuration - User needs to provide the operation for download or upload the file.
  • Operation - Operation has two options -
      Get Files - User is required to download the file from server to local.
      Put Files - User is required to upload the file from local to server.

      If user select protocol : AWS S3 HTTPS

  • Character Set - Contains a drop-down of character encoding values. Used simply to denote the encoding system that will be used while creating the Output file to denote characters of data. Drop down lists values like : UTF8, US-ASCII, ISO-8859-1 etc.
  • Fail On Error - This property accepts Boolean values True and False. The Fail property when True, If user need to fail the component. When False, If user need to pass the component on job execution.
  • Batch - Batch simply accepts a numeric number starting from 0 to maximum 99 and signifies the batch this component will execute in. By default this is 0.

Validations

The FTP components applies validations to the mandatory fields. Upon placing the FTP component on job canvas for the first time (from component palette), the component shows up a warning icon as mandatory properties are not provided.

The properties window also displays error icon on mandatory fields if it has an incorrect value. The error icon is displayed on the tab as well, if any of the field within the tab has some error.

If the properties window has some error even after user visit's it once, then the warning icon on the FTP component on the job canvas changes to error icon. This error icon is removed only when all the mandatory fields are supplied with correct values.
